8-STEP PROCESS IMPLEMENTATION

✅ STEP 1-2: Data Loading
   - Load maintenance activities
   - Load project activities
   - Combine all sources

✅ STEP 3: Recent Visit Exclusion
   - Check visit history (configurable months: default 3)
   - Remove recently visited sites
   - Preserve remaining sites for assignment

✅ STEP 4: Critical Site Exclusion
   - Load critical sites list
   - Identify high-risk rooftop sites
   - Exclude from schedule

✅ STEP 5: Activity Harmonization
   - Combine maintenance and project activities
   - Create unified activity list
   - Add metadata and classifications

✅ STEP 6: Site Assignment to Teams
   - Assign 2 sites per team member
   - Assign 4 sites per safety influencer
   - Balance workload across teams

✅ STEP 7: Team Formation
   - Create team pairs/trios
   - Technical lead + non-technical resources
   - Rotate team composition monthly

✅ STEP 8: Time Limit Setting
   - Maintenance activities: Use original schedule
   - Project activities: Mark TBC for coordination
   - Set visit timelines

INPUT REQUIREMENTS
================================================================================

Excel File Format (.xlsx)

REQUIRED SHEETS:
├─ Sheet: "maintenance"
│  ├─ site_id (text)
│  ├─ site_name (text)
│  ├─ nearest_hi (number)
│  ├─ tower_type (text: "Rooftop" or "Greenfield")
│  ├─ activity_type (should be "maintenance")
│  └─ time_limit (text, e.g., "09:00-17:00")
│
├─ Sheet: "project"
│  ├─ site_id (text)
│  ├─ site_name (text)
│  ├─ project_name (text)
│  ├─ department (text)
│  ├─ activity_type (should be "project")
│  └─ time_limit (optional)

OPTIONAL SHEETS:
├─ Sheet: "visit_history"
│  ├─ site_id (text)
│  ├─ visit_date (date: YYYY-MM-DD format)
│  └─ team_member (text)
│
└─ Sheet: "critical_sites"
   ├─ site_id (text)
   ├─ site_name (text)
   ├─ tower_type (text)
   ├─ risk_level (text: "high", "medium", "low")
   └─ reason (text)

Example: See data/sample_schedule.xlsx
